# Microservices Architecture Skill

## Instructions

When designing microservices architectures:

1. **Service Boundaries**: Define clear domain boundaries
2. **Communication**: Choose appropriate sync/async patterns
3. **Data Management**: Implement per-service databases
4. **Resilience**: Add circuit breakers and retries
5. **Observability**: Implement distributed tracing

## Patterns

- [Event-Driven Architecture](./patterns/event-driven-architecture.md)
- [Microservices Patterns](./patterns/microservices-patterns.md)

## Guidelines

- Design for failure and resilience
- Implement proper service discovery
- Use API gateways for edge services
- Maintain backward compatibility
- Implement comprehensive monitoring
