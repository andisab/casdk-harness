<validation_checks>
## Validation Checks

### 1. Schema Compliance

Check against test_suite.schema.json requirements:

**Suite level:**
- [ ] `name` is present and non-empty
- [ ] `agent_name` is present and non-empty
- [ ] `test_cases` array has 1+ items

**Each test case:**
- [ ] `id` is unique, lowercase with hyphens
- [ ] `prompt` is present, min 10 characters
- [ ] `expected_behavior` is present
- [ ] `validation.type` is valid enum value
- [ ] `validation.criteria` is non-empty

**Validation types allowed:**
- exact, contains, regex, llm_judge
- code, code_syntax, code_llm

### 2. Coverage Analysis

**Competency coverage:**
```
For each competency in eval_criteria.yaml:
  - Check if ANY test has metadata.source_competency matching
  - OR if test tags include competency name
  - OR if test prompt references competency concept
```

**Edge case coverage:**
```
For each edge_case in eval_criteria.yaml:
  - Check if test with metadata.source_type == "edge_case"
  - AND scenario matches
```

**Mistake coverage:**
```
For each common_mistake in eval_criteria.yaml:
  - Check if test with metadata.source_type == "common_mistake"
  - OR tags include ["negative"]
```

### 3. Quality Metrics

**Difficulty balance:**
- Count tests by difficulty tag
- Compare to target: 40% basic, 40% intermediate, 20% advanced
- Allow +/- 10% variance

**Validation type appropriateness:**
- Code tests should use code/code_llm/code_syntax
- Text tests should use llm_judge/contains
- Flag mismatches

**Timeout reasonableness:**
- Basic tests: 60-300s
- Intermediate: 120-600s
- Advanced: 300-900s
- Flag outliers

### 4. Issue Detection

**Critical issues (fail validation):**
- Missing required fields
- Invalid validation types
- Duplicate test IDs
- Empty test_cases array

**Warnings (pass with warnings):**
- Missing competency coverage
- Difficulty imbalance
- Missing metadata

**Info (always pass):**
- Timeout variations
- Optional field absence
</validation_checks>

<coverage_calculation>
## Coverage Calculation

### Competency Coverage

```python
def calculate_competency_coverage(tests, competencies):
    covered = set()
    for test in tests:
        # Check metadata
        if test.metadata.source_competency:
            covered.add(test.metadata.source_competency)
        # Check tags
        for comp in competencies:
            if comp.name.lower() in str(test.tags).lower():
                covered.add(comp.name)
            if comp.name.lower() in test.prompt.lower():
                covered.add(comp.name)

    return len(covered) / len(competencies) * 100
```

### Edge Case Coverage

```python
def calculate_edge_case_coverage(tests, edge_cases):
    edge_tests = [t for t in tests
                  if t.metadata.source_type == "edge_case"
                  or "edge_case" in t.tags]
    return len(edge_tests) / len(edge_cases) * 100 if edge_cases else 100
```

### Mistake Coverage

```python
def calculate_mistake_coverage(tests, mistakes):
    neg_tests = [t for t in tests
                 if t.metadata.source_type == "common_mistake"
                 or "negative" in t.tags]
    return len(neg_tests) / len(mistakes) * 100 if mistakes else 100
```
</coverage_calculation>
