# Documentation Skill

## Instructions

When creating documentation:

1. **Audience**: Identify target readers
2. **Structure**: Organize content logically
3. **Examples**: Provide clear, working examples
4. **Clarity**: Use simple, precise language
5. **Maintenance**: Keep docs up-to-date

## Workflows

- [Documentation Generation](./workflows/documentation-generation.md)

## Guidelines

- Start with quickstart guide
- Include API references
- Add troubleshooting section
- Provide code examples
- Use diagrams for complex concepts
