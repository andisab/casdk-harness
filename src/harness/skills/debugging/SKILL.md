# Debugging Skill

## Instructions

When debugging issues:

1. **Reproduce**: Consistently reproduce the issue
2. **Isolate**: Narrow down the problem area
3. **Investigate**: Use debugging tools and logging
4. **Hypothesize**: Form testable theories
5. **Fix & Verify**: Apply fixes and confirm resolution

## Workflows

- [Bug Fix Debugging](./workflows/bug-fix-debugging.md)

## Guidelines

- Use systematic approach, not random changes
- Add logging before debugging
- Check recent changes first
- Use debugger and profiling tools
- Document root cause and fix
